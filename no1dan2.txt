sebutkan dan jelaskan perintah perintah dasar yang digunakan dalam git
git init Perintah ini digunakan untuk membuat repositori Git baru di direktori kerja saat ini. Repositori ini akan digunakan untuk melacak perubahan pada proyek Anda.
git clone <url> Perintah ini digunakan untuk mengkloning repositori Git yang sudah ada dari server ke direktori lokal Anda. URL di sini adalah alamat repositori yang ingin Anda kloning.
git add <file> Perintah ini digunakan untuk menambahkan perubahan pada file tertentu ke "staging area". Staging area adalah area persiapan sebelum perubahan disimpan secara permanen dalam repositori.
git commit -m "<pesan commit>" Perintah ini digunakan untuk menyimpan perubahan yang sudah di-staging ke dalam repositori. Pesan commit berisi deskripsi singkat tentang perubahan yang telah Anda lakukan.
git status Perintah ini digunakan untuk melihat status repositori saat ini, termasuk file yang sudah diubah, yang sudah di-staging, dan yang belum diikuti oleh Git.
git log Perintah ini digunakan untuk melihat daftar riwayat commit di repositori. Ini akan menampilkan daftar commit dengan informasi waktu, penulis, dan pesan commit.
git push Perintah ini digunakan untuk mengirimkan perubahan yang sudah di-commit dari repositori lokal ke repositori yang berada di server. Biasanya digunakan untuk mengunggah perubahan ke repositori pusat atau berbagi perubahan dengan tim.
git pull Perintah ini digunakan untuk mengambil perubahan dari repositori yang berada di server dan memperbarui repositori lokal Anda dengan perubahan terbaru.
git branch Perintah ini digunakan untuk melihat daftar branch yang ada di repositori Anda. Branch adalah cabang independen dalam repositori yang memungkinkan Anda untuk bekerja pada fitur atau perbaikan tertentu tanpa mengganggu branch utama (biasanya disebut "master" atau "main").
git checkout <branch> Perintah ini digunakan untuk beralih ke branch lain. Anda dapat beralih ke branch yang sudah ada atau membuat branch baru dengan perintah ini.
git merge <branch> Perintah ini digunakan untuk menggabungkan perubahan dari branch lain ke branch saat ini. Biasanya, Anda akan menggunakan ini setelah bekerja di branch fitur dan ingin menggabungkannya kembali ke branch utama.
git remote Perintah ini digunakan untuk melihat daftar remote repository yang sudah diatur untuk repositori lokal Anda. Remote repository adalah repositori di server eksternal yang digunakan untuk berkolaborasi atau menyimpan cadangan proyek Anda.
git fetch Perintah ini digunakan untuk mengambil perubahan dari remote repository tetapi tidak menggabungkannya dengan branch lokal Anda. Anda perlu menggunakan perintah git merge untuk menggabungkan perubahan tersebut setelah git fetch.
git diff Perintah ini digunakan untuk melihat perbedaan antara file yang sudah diubah dan versi sebelumnya yang ada di repositori.
git rm <file> Perintah ini digunakan untuk menghapus file dari repositori Anda dan menghapusnya dari staging area.
2. sebutkan dan jelaskan perintah perintah dalam berkolaborasi menggunakan git
git clone <url> Sebelum berkolaborasi, Anda perlu mendapatkan salinan (clone) dari remote repository. Perintah ini mengkloning seluruh repositori dari server ke direktori lokal Anda. URL adalah alamat remote repository yang ingin Anda klone.
git pull Perintah ini digunakan untuk mengambil perubahan terbaru dari remote repository dan memperbarui repositori lokal Anda dengan perubahan tersebut. Ini berguna jika ada perubahan yang dilakukan oleh anggota tim lainnya dan Anda ingin memperbarui repositori lokal Anda sebelum melanjutkan pekerjaan Anda.
git push Setelah Anda melakukan perubahan di repositori lokal dan ingin berbagi perubahan tersebut dengan tim, Anda menggunakan perintah ini untuk mengunggah (push) perubahan ke remote repository. Misalnya: git push origin <branch> untuk mengunggah perubahan dari branch saat ini ke branch yang sesuai di remote repository.
git remote -v Perintah ini digunakan untuk melihat daftar remote repository yang sudah diatur untuk repositori lokal Anda. Ini akan menampilkan alamat URL remote repository yang terkait dengan "origin" dan "fetch" (ambil) dan "push" (unggah).
git fetch Perintah ini mengambil perubahan dari remote repository tetapi tidak langsung menggabungkannya dengan branch lokal Anda. Perintah ini berguna untuk melihat perubahan yang ada di remote repository sebelum menggabungkannya dengan repositori lokal menggunakan git merge atau git rebase.
git merge <branch> Ketika Anda ingin menggabungkan perubahan dari branch lain ke branch saat ini, Anda dapat menggunakan perintah ini. Misalnya, jika Anda telah menyelesaikan fitur di branch "feature-branch" dan ingin menggabungkannya ke branch utama (biasanya "master" atau "main"), Anda dapat menggunakan git merge feature-branch.
git pull --rebase Ini adalah cara alternatif untuk mendapatkan perubahan terbaru dari remote repository. Perintah ini mengambil perubahan dan menggabungkannya dengan repositori lokal Anda dengan melakukan rebase (pengalihan garis dasar) daripada melakukan merge. Ini membantu menjaga sejarah commit menjadi lebih bersih.
git push --force Biasanya, ketika Anda melakukan git push, Git hanya akan menerima perubahan yang cepat (fast-forward) untuk memastikan sejarah commit tetap linier. Namun, ada situasi tertentu di mana Anda mungkin perlu memaksa mengunggah perubahan yang menimpa sejarah commit di remote repository. Gunakan git push --force dengan hati-hati karena dapat menyebabkan masalah jika tidak digunakan dengan benar.
git branch -r Perintah ini digunakan untuk melihat daftar branch yang ada di remote repository. Ini membantu Anda melihat cabang-cabang yang dibuat oleh anggota tim lain di remote repository.
git checkout -b <branch> Jika Anda ingin bekerja di branch baru, Anda dapat menggunakan perintah ini untuk membuat branch baru dan beralih ke branch tersebut dalam satu langkah.
git log <remote>/<branch> Perintah ini digunakan untuk melihat riwayat commit di branch yang ada di remote repository tanpa harus mengambil perubahan tersebut ke repositori lokal Anda.
git remote add <name> <url> Jika Anda ingin menambahkan remote repository baru, Anda dapat menggunakan perintah ini. <name> adalah alias untuk remote repository sehingga Anda dapat merujukkannya dengan nama itu daripada URL lengkap.
